# 内存虚拟化 & ASLR

## 核心思想

OS 给每个进程一个**私有的虚拟地址空间**，进程以为自己独占整块内存，实际上 OS + MMU 在背后把虚拟地址翻译成不同的物理地址。

```
进程 A 虚拟空间          进程 B 虚拟空间
0x0000...                0x0000...
0x5555... (p)            0x5555... (p)   ← 虚拟地址相同
0xFFFF...                0xFFFF...
        ↓ MMU 翻译               ↓ MMU 翻译
    物理地址 0x1a2b...        物理地址 0x9f3c...  ← 物理地址不同
```

---

## 页（Page）

OS 以**页**为单位管理内存，不是逐字节管理。

- 虚拟内存的块：**Page（页）**
- 物理内存的块：**Frame（帧）**
- 默认大小：**4KB**（`getconf PAGE_SIZE`）
- 页表记录"虚拟页 → 物理帧"的映射关系，每个进程私有

### 地址翻译

MMU 把虚拟地址拆成两部分：

```
虚拟地址
┌──────────────────────┬────────────┐
│   页号 (VPN)          │  页内偏移  │
└──────────────────────┴────────────┘
         │                    │
         ↓ 查页表              │（直接复用）
      帧号 (PFN)               │
         └─────────────────────┘
                  ↓
             物理地址
```

---

## 内存隔离

进程无法访问其他进程的内存：

- 每个进程有自己的页表，由 OS 维护
- 进程切换时，OS 同时切换页表
- 访问未映射的虚拟地址 → MMU 触发 **Page Fault** → OS 发送 SIGSEGV → **Segmentation Fault**
- 进程的页表里不存在指向其他进程物理内存的条目，无法越界访问

---

## ASLR（Address Space Layout Randomization）

每次程序启动，OS 把堆、栈、代码段的起始地址**随机偏移**，防止攻击者预测内存布局（ROP 攻击等）。

```bash
# 查看当前 ASLR 状态
cat /proc/sys/kernel/randomize_va_space
# 0 = 关闭，1 = 部分随机，2 = 完全随机（默认）

# 关闭 ASLR（演示用）
echo 0 | sudo tee /proc/sys/kernel/randomize_va_space

# 恢复
echo 2 | sudo tee /proc/sys/kernel/randomize_va_space
```

### 程序如何在 ASLR 下正常运行

ASLR 只随机化**基地址**，程序内部的**相对关系**不变。

**自己的代码/变量** → RIP 相对寻址：
```asm
leaq .LC6(%rip), %rdi   ; 从当前指令偏移，不写死绝对地址
```

**外部库函数（printf 等）** → PLT + GOT：
- 程序启动时动态链接器把库函数真实地址填进 GOT
- 运行时通过 PLT 跳板查 GOT 找到真实地址

## 用户态的一切都是假象

| 假象 | 真相 | 维护者 |
|------|------|--------|
| 独占 CPU | 时间片轮流切换 | OS 调度器 |
| 独占内存 | 虚拟地址空间 + 页表隔离 | OS + MMU |
| 内存连续 | 物理内存可能是碎片 | OS 内存管理 |
| 库函数地址固定 | 动态链接器运行时填入 | OS + `ld.so` |

OS 的核心工作：**虚拟化硬件资源，让每个程序以为自己独占机器。**
