# fork 流程 & execlp

## fork()

`fork()` 调用一次，**返回两次**：

```c
pid = fork();
```

| 返回值 | 谁在执行 | 含义 |
|--------|----------|------|
| `< 0` | 父进程 | fork 失败 |
| `== 0` | 子进程 | 我是新创建的子进程 |
| `> 0` | 父进程 | 返回值是子进程的 PID |

### 时间线

```
父进程                          子进程
  │                                │
  │── fork() ──────────────→ 诞生（内存完全复制）
  │                                │
  │ waitpid(pid)              execlp(buf, buf, NULL)
  │ （阻塞等待）               （内存替换为新程序）
  │                                │
  │ ←──────────────────── 子进程结束
  │
  │ 打印提示符，等待下一条命令
```

### fork 后子进程的特点

- 父进程内存的**完整副本**（代码、数据、栈、堆）
- 继承父进程的文件描述符
- 有自己独立的 PID
- 独立的虚拟地址空间，修改变量互不影响

## execlp()

```c
int execlp(const char *file, const char *arg0, ... /*, (char *)0 */);
```

### 参数

| 参数 | 含义 |
|------|------|
| `file` | 要执行的程序名，在 `$PATH` 里自动搜索 |
| `arg0` | 传给新程序的 `argv[0]`，约定填程序名本身 |
| `...` | 后续参数，以 `(char *)0`（NULL）结尾 |

### 做了什么

把当前进程的**整个内存镜像替换**为新程序：

```
调用前（子进程）          调用后（子进程）
┌──────────┐             ┌──────────┐
│ shell代码 │  execlp()  │  ls 代码  │
│ shell数据 │ ─────────→ │  ls 数据  │
│ shell栈  │             │  ls 栈   │
└──────────┘             └──────────┘
```

- PID 不变
- `execlp` 成功后**永远不返回**，原代码已不存在
- 只有失败时才返回（返回 -1），后续的 `err_ret` / `exit(127)` 才会执行

### exec 家族后缀规则

| 后缀 | 含义 |
|------|------|
| `l` | 参数逐个列出（list） |
| `v` | 参数用数组传（vector） |
| `p` | 自动搜索 `$PATH` |
| `e` | 自定义环境变量 |

## shell1.c 完整流程

```c
while (fgets(buf, ...)) {       // 读用户输入
    if ((pid = fork()) == 0) {  // 子进程
        execlp(buf, buf, NULL); // 执行命令
        exit(127);              // execlp 失败才到这
    }
    waitpid(pid, &status, 0);   // 父进程等待子进程结束
    printf("%% ");              // 打印提示符
}
```

### waitpid 的作用

防止**僵尸进程（Zombie Process）**：子进程结束后，其 PCB（进程控制块）仍留在内核，等父进程调用 `wait`/`waitpid` 来"收尸"。若父进程不等待，子进程变成僵尸，占用系统资源。

## shell2.c 新增：信号处理

```c
signal(SIGINT, sig_int);   // 注册 Ctrl+C 的处理函数

void sig_int(int signo) {
    printf("interrupt\n%% ");  // 不退出，打印提示符继续
}
```

没有这个处理，Ctrl+C（SIGINT）会直接终止整个 shell。注册后，Ctrl+C 只触发 `sig_int`，shell 继续运行。
